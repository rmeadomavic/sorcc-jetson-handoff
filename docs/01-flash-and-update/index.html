Section 01 · ~90 min

Flash & update

This is the hard part. You are taking the Jetson from a stock-from-NVIDIA dev kit, which ships with old firmware that cannot boot JetPack 6, all the way to a fully updated JetPack 6.x system in MAXN Super power mode. Do these in order. Do not skip ahead.

How to know you can skip a step

  • Skip QSPI update if you have already booted JetPack 6.x on this exact board before. Verify with cat /etc/nv_tegra_release. If it starts with # R36, you do not need QSPI.
  • Skip flash and first boot if your Jetson already has a working JetPack 6.x install you do not want to wipe.
  • You probably cannot skip MAXN Super unless you specifically remember setting it. Check with sudo nvpmodel -q.
Stuck? The single most common failure: the QSPI updater package name has a lowercase L, not a number 1. nvidia-l4t-... not nvidia-14t-.... If apt says "Unable to locate package", check this first. Troubleshooting covers every other failure mode anyone has hit.
