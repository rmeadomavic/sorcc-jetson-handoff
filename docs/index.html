Read these first · three known traps

If you only read three things in this repo, read these. Each one has bitten a real student.

Trap 01 QSPI before JetPack 6.

Brand-new Orin Nano dev kits cannot boot a JetPack 6 SD card until you flash QSPI from a JetPack 5.1.3 card first. The board powers on and stares at you with a blank screen. See Section 01.01.

Trap 02 Lowercase L, not number 1.

The package name is nvidia-l4t-jetson-orin-nano-qspi-updater. That is l4t, not 14t. apt fails with "Unable to locate package" on a typo, with no other clue. Tab-complete to be safe.

Trap 03 QSPI updater is single-shot per card.

Reusing the same JP 5.1.3 card on a second Jetson? Add --reinstall or the post-install hook will be a silent no-op and the QSPI stays untouched.

Heads up No em dashes, no shortcuts.

Every command in every doc is shown in full. Copy buttons on terminal blocks strip the $ prompt for you. Use them. Typos are the #1 cause of "it didn't work."

Where should you start?

Has this Jetson ever booted JetPack 6.x?
If no → start at Section 01. Check with cat /etc/nv_tegra_release (look for # R36).
Is Docker plus the NVIDIA container toolkit installed?
If no → Section 02 base bootstrap.
Are you wiring a Pixhawk flight controller?
If yes → Section 03. Otherwise skip it.
Which apps do you want?
Hydra Detect · Ollama · ComfyUI — pick any. See Section 04.

Hardware scope

This guide covers exactly one configuration. Other Jetsons (Nano 4GB, NX, AGX) might mostly work but are not tested. The 4GB Nano in particular will run out of memory in the apps section.
