Section 02 · ~20 min

Base system

After this section, your Jetson has Docker, the NVIDIA container runtime, the right group permissions, and (optionally) Tailscale for remote access. Every app in Section 04 assumes you have finished this section.

Get the repo on your Jetson

Skip this block if you already cloned the repo back on the landing page. Otherwise, do it now, everything in Section 02 (and Section 04) runs from inside this checkout:

Shell · sorcc@jetson
$ sudo apt-get install -y git
$ git clone https://github.com/rmeadomavic/sorcc-jetson-handoff.git ~/sorcc-jetson-handoff
$ cd ~/sorcc-jetson-handoff

Quick path

From the repo root, on your Jetson:

Shell · sorcc@jetson
$ bash 02-base-system/01-base-bootstrap.sh

That installs Docker, NVIDIA Container Toolkit, adds you to the docker and dialout groups, and pulls the dustynv/l4t-pytorch:r36.4.0 base image (about 6 GB). The script is idempotent, safe to re-run, and will not double-install or break existing setup.

After it finishes, log out and back in (or run newgrp docker) so the group changes apply. Then verify:

Shell · sorcc@jetson
$ bash 02-base-system/verify.sh

If verify is green, continue to the optional Tailscale step or skip ahead to Section 04 · Apps.

Files in this section

  • 01-base-bootstrap.sh, does everything in the table above.
  • 02-tailscale.md, install and configure Tailscale for remote access (optional).
  • verify.sh, checks all of the above worked.

What this section does NOT do

  • No app installs (Hydra, Ollama, ComfyUI). Those are in Section 04.
  • No flight controller wiring. That is Section 03.
  • No platform-specific config (callsign, hostname, etc). The original golden image had a platform-setup.sh for that. For a personal Jetson, you usually do not need per-unit identity.
