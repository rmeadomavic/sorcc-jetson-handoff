Section 02 · Step 01 · ~20 min

Base bootstrap

One script. Six steps. It brings a fresh JetPack 6 Jetson Orin Nano up to a state where every app in Section 04 can run. Safe to re-run, each step checks for existing state before acting.

Heads up The base image pull in Step 04 is about 6 GB. On a hotspot or slow link, plan for 30 minutes or more on that step alone. The rest of the script finishes in a few minutes.

Run it

From the repo root, on the Jetson:

Shell · sorcc@jetson
$ bash 02-base-system/01-base-bootstrap.sh

What the script does, step by step

01

apt update and upgrade.

Brings every system package up to date. Without this, later steps can fail on stale package indexes or hold-back conflicts. Runs non-interactively so it does not stop on a config-file prompt.

Shell · sorcc@jetson
$ sudo apt-get update
$ sudo DEBIAN_FRONTEND=noninteractive apt-get upgrade -y
02

Install Docker CE.

Container runtime. JetPack 6 already has NVIDIA's apt repo configured, so this is a simple apt-get install docker.io followed by enabling the service. If Docker is already installed, the script prints the existing version and skips.

Shell · sorcc@jetson
$ sudo apt-get install -y docker.io
$ sudo systemctl enable --now docker
03

Install NVIDIA Container Toolkit.

This is the bridge that lets containers see the GPU. Without it, every Hydra, Ollama, and ComfyUI run will fail with no CUDA device. Installing the package and restarting Docker is enough, no manual daemon.json editing required on JetPack 6.

Shell · sorcc@jetson
$ sudo apt-get install -y nvidia-container-toolkit
$ sudo systemctl restart docker
04

Pull the base Docker image.

The image is dustynv/l4t-pytorch:r36.4.0, about 6 GB. It carries CUDA, PyTorch, and TensorRT pre-built for L4T R36.4. Every app in Section 04 builds on top of it. If the image is already present locally, the script skips the pull.

Shell · sorcc@jetson
$ sudo docker pull dustynv/l4t-pytorch:r36.4.0
05

Add your user to docker and dialout groups.

docker means you can run containers without sudo. dialout means you can read and write the serial port a Pixhawk shows up on (/dev/ttyACM0). After this step you must log out and back in, or run newgrp docker, before the new groups apply.

06

Create an 8 GB swap file.

The 8 GB shared RAM on the Orin Nano fills up fast once a model is loaded and a container is running. An 8 GB swap file at /swapfile keeps the system from being OOM-killed under load. The script writes an /etc/fstab entry so swap survives reboot.

Shell · sorcc@jetson
$ sudo fallocate -l 8G /swapfile
$ sudo chmod 600 /swapfile
$ sudo mkswap /swapfile
$ sudo swapon /swapfile
$ echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

After the script finishes

Log out and back in (or run newgrp docker) so the group additions in Step 05 take effect. Then run the project's preflight to confirm the system is ready for apps.

Shell · sorcc@jetson
$ bash scripts/preflight.sh

Validate

The section ships a deeper check that actually runs a container and confirms CUDA is visible inside it. Run this before you move on:

Shell · sorcc@jetson
$ bash 02-base-system/verify.sh

Green across the board means the base system is ready for any app in Section 04.
