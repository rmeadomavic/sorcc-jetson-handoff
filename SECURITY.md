# Security Policy

## Supported versions

The latest commit on `main` is supported. Earlier commits or tags receive no fixes.

## Reporting a vulnerability

Email **kyle.adomavicius@gmail.com** with:
- a description of the issue
- reproduction steps if you have them
- the affected version, tag, or commit hash

I will acknowledge within 7 days and aim to issue a fix or mitigation within 30 days for confirmed issues.

Please do **not** open a public GitHub issue for security reports — use email so the fix can ship before disclosure.

## Scope

This policy covers code in this repository. It does not cover:
- Third-party dependencies (report those upstream).
- Hardware vulnerabilities in any device this project interacts with.
- Operating system or firmware on supporting hardware.
